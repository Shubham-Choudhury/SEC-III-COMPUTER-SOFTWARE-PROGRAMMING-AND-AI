## 1. The Six Phases of SDLC

Every software project generally passes through these six phases:

![Six Phases of SDLC]({{ '/assets/images/Six-Phases-of-SDLC.png' | relative_url }})

## **Phase 1:** Requirement Analysis

Requirement Analysis is the process of collecting, analyzing, documenting, and validating the needs and expectations of users.

### # Objectives

- Understand customer requirements.
- Identify business needs.
- Determine project scope.
- Estimate time and cost.

## **Phase 2:** System Design

System Design is the process of converting requirements into a blueprint for building the software.

### # Objectives

- Design system architecture.
- Design databases.
- Design user interfaces.
- Select technologies.

## **Phase 3:** Implementation (Coding)

Implementation is the process of converting the design into source code using a programming language.

### # Objectives

- Develop software modules.
- Follow coding standards.
- Produce efficient and maintainable code.

## **Phase 4:** Testing

Testing is the process of verifying that the software works correctly and meets user requirements.

### # Objectives

- Find bugs.
- Improve software quality.
- Verify functionality.
- Ensure reliability.

### # Types of Testing

1. Unit Testing:

   Tests individual functions or modules.

   Example: Testing only the login function.

2. Integration Testing

   Tests interactions between modules.

   Example: Login + Database connection.

3. System Testing

   Tests the complete software system.

4. Acceptance Testing

   Performed by customers to determine whether the software meets their requirements.

## **Phase 5:** Deployment

Deployment is the process of installing and making the software available for users.

### # Objectives

- Install the software.
- Configure the environment.
- Train users.
- Release the software.

## **Phase 6:** Maintenance

Maintenance is the process of modifying and updating software after deployment.

### # Why is Maintenance Needed?

- Bugs are discovered.
- New features are requested.
- Operating systems change.
- Security vulnerabilities are found.
- User requirements evolve.

### # Types of Maintenance

1. Corrective Maintenance

   Fixes software errors.

   Example: Correcting an incorrect CGPA calculation.

2. Adaptive Maintenance

   Modifies software to work in a changed environment.

   Example: Updating software to support a new version of Windows.

3. Perfective Maintenance

   Adds new features or improves performance.

   Example: Adding online fee payment.

4. Preventive Maintenance

   Improves software to reduce the likelihood of future problems.

   Example: Refactoring code to improve maintainability.
