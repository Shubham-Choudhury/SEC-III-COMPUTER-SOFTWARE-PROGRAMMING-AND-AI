## 1. What is Software Development?

Software Development is the process of designing, creating, testing, deploying, and maintaining software applications to solve specific problems or fulfill user requirements.

It includes all activities required to build software from the initial idea to the final product.

## 3. What is Software Engineering?

Software Engineering is the application of engineering principles, methods, and tools to the systematic development, operation, and maintenance of software.

### # Software Development vs Software Engineering

| Software Development            | Software Engineering                                                                   |
| ------------------------------- | -------------------------------------------------------------------------------------- |
| Focuses on building software    | Focuses on the complete engineering process                                            |
| Coding is a major activity      | Includes planning, design, testing, maintenance, documentation, and project management |
| Suitable for small projects     | Suitable for medium and large projects                                                 |
| May not follow formal processes | Follows systematic engineering practices                                               |

### # Need for Software Engineering

Software engineering is important because it helps:

- Develop reliable software.
- Reduce development cost.
- Deliver software on time.
- Improve software quality.
- Simplify maintenance.
- Enhance security.
- Increase customer satisfaction.
- Support teamwork in large projects.

### # Problems Without Software Engineering

If software is developed without proper planning:

- Poor quality code.
- Frequent errors.
- Delayed project completion.
- Budget overruns.
- Difficult maintenance.
- Poor user experience.
- Security vulnerabilities.

### # Characteristics of Good Software

Good software should satisfy user needs and maintain high quality.

The main characteristics include:

1. Correctness: The software should produce accurate and expected results according to user requirements.
2. Reliability: Software should perform consistently without failures under expected operating conditions.
3. Efficiency: Software should use system resources effectively, such as CPU time, memory, storage, and network bandwidth.
4. Usability: Software should be easy to learn, understand, and use.
5. Maintainability: Software should be easy to modify, update, and fix.
6. Portability: Software should run on different platforms with minimal changes.
7. Security: Software should protect data from unauthorized access and cyber threats.
8. Scalability: Software should handle increasing numbers of users or larger amounts of data without significant performance degradation.

### # Software Quality Attributes

Software quality refers to how well software satisfies specified requirements and user expectations.

Common quality attributes include:

| Attribute       | Description                                |
| --------------- | ------------------------------------------ |
| Correctness     | Produces expected results                  |

| Efficiency      | Uses resources effectively                 |
| Usability       | Easy to use                                |
| Maintainability | Easy to update and repair                  |
| Portability     | Works across platforms                     |
| Security        | Protects data and systems                  |
| Scalability     | Supports growth                            |
| Reusability     | Components can be reused in other projects |
| Testability     | Easy to test for defects                   |

## 4. Software Engineer

A Software Engineer is a professional who applies engineering principles to design, develop, test, deploy, and maintain software systems.

### # Responsibilities of a Software Engineer

- Gather user requirements.
- Design software architecture.
- Write program code.
- Test software.
- Fix defects.
- Deploy software.
- Maintain and improve applications.
- Document the system.
- Collaborate with team members.
