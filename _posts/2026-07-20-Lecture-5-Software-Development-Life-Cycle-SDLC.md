## 1. What is SDLC?

SDLC is a systematic process for developing high-quality software efficiently and reliably.

### # Objectives of SDLC

The main objectives are:

- Deliver high-quality software.
- Meet customer requirements.
- Complete projects on time.
- Reduce development costs.
- Improve project management.
- Minimize risks.
- Simplify maintenance.
- Increase customer satisfaction.

### # Advantages of SDLC

- Better planning.
- Improved communication.
- Easier project management.
- Reduced errors.
- Higher software quality.
- Better documentation.
- Easier maintenance.
- Faster development in later phases.
- Predictable project timelines.

## 3. SDLC Models

An SDLC model is a strategy or framework that defines how the development phases are organized and executed.

| Model             | Description                           |
| ----------------- | ------------------------------------- |
| Waterfall Model   | Sequential approach                   |
| V-Model           | Verification and Validation           |
| Incremental Model | Developed in smaller parts            |
| Spiral Model      | Risk-driven development               |
| Agile Model       | Iterative and flexible                |
| Prototype Model   | Early working model for user feedback |

## I. Waterfall Model

In waterfall model, development progresses through a series of phases, where each phase must be completed before the next one begins.

### # Waterfall Model Diagram

![Waterfall Model Diagram]({{ '/assets/images/Waterfall-Model-Diagram.jpg' | relative_url }})

### # Characteristics of the Waterfall Model

- Sequential process.
- Simple and easy to understand.
- Well-documented.
- Each phase has specific deliverables.
- Minimal overlap between phases.

### # Advantages of the Waterfall Model

- Easy to manage.
- Clear milestones.
- Good documentation.
- Suitable for small, stable projects.
- Easy to estimate cost and schedule.

### # Limitations of the Waterfall Model

- Difficult to accommodate changing requirements.
- Customer feedback is usually received late.
- Errors discovered in later phases can be expensive to fix.
- Not ideal for large or rapidly changing projects.

## II. Agile Model

The Agile Model is an iterative and flexible software development approach that delivers software in small increments and incorporates continuous customer feedback.

### # Agile Development Cycle

![Agile Development Cycle]({{ '/assets/images/Agile-Development-Stages.png' | relative_url }})

### # Characteristics of Agile

- Customer involvement throughout development.
- Frequent software releases.
- Adaptable to changing requirements.
- Team collaboration.
- Continuous testing and improvement.

### # Advantages of Agile

- Faster delivery of working software.
- Easy to incorporate requirement changes.
- Higher customer satisfaction.
- Continuous feedback.
- Early detection of problems.

### # Limitations of Agile

- Requires experienced and collaborative teams.
- Less predictable cost and timeline.
- Documentation may be lighter than traditional models.
- Not suitable for every project.
